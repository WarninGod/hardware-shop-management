  - Quantity: 5
  - Total: 120 Ã— 5 = 600.00
  - Cost: 80 Ã— 5 = 400.00
  - Profit: 600 - 400 = 200.00
  - Stock reduced: 50 - 5 = 45

Errors:
- Stock not available: "Insufficient stock. Available: 40, Requested: 50"
- Invalid quantity: "Quantity must be a positive number"
- Product not found: "Product not found"

Run these commands in sequence to test the entire workflow:

Step 1: Create Vendor
curl -X POST http://localhost:3000/vendors ^
  -H "Content-Type: application/json" ^
  -d "{\"name\":\"Test Vendor\",\"phone\":\"9876543210\"}"

Step 2: Create Product (use vendor_id from Step 1 response)
curl -X POST http://localhost:3000/products ^
  -H "Content-Type: application/json" ^
  -d "{\"name\":\"Test Product\",\"category\":\"Test\",\"vendor_id\":1,\"cost_price\":100,\"selling_price\":150,\"stock_quantity\":100}"

Step 3: Record Sale (use product_id from Step 2 response)
curl -X POST http://localhost:3000/sales ^
  -H "Content-Type: application/json" ^
  -d "{\"product_id\":1,\"quantity\":10}"

Step 4: Check Summary
curl http://localhost:3000/reports/summary

Step 5: Check Product Profit
curl http://localhost:3000/reports/product-profit

Step 6: Check Vendor Profit
curl http://localhost:3000/reports/vendor-profit

Expected Results:
  - 1 sale recorded
  - 10 units sold
  - Revenue: 1500 (150 Ã— 10)
  - Profit: 500 ((150-100) Ã— 10)
  - Stock reduced to 90 (100 - 10)

1. Always use correct Content-Type: application/json

2. Verify required fields in request body

3. Check JSON syntax (use https://jsonlint.com if needed)

4. Test error cases:
   - Invalid vendor_id
   - Stock exceeding available quantity
   - Selling price < cost price
   - Negative quantities/prices

5. Use browser DevTools (F12) to see API calls
   - Network tab shows requests/responses
   - Console tab shows errors

6. Check server console for detailed error logs

7. Test with curl first, then use Postman for complex scenarios
